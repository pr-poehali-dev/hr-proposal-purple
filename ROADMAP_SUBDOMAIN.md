# Настройка поддомена roadmap

## Как подключить поддомен roadmap.* к проекту

### Автоматическое определение поддомена

Приложение автоматически определяет, открыто ли оно на поддомене `roadmap.*` и показывает соответствующую страницу:

- **Основной домен** (example.com) → Показывается страница подбора команды (Index)
- **Поддомен roadmap** (roadmap.example.com) → Показывается дорожная карта проекта (Roadmap)

### Настройка DNS

Чтобы поддомен работал, добавьте в DNS вашего домена:

```
Тип: CNAME
Имя: roadmap
Значение: ваш-основной-домен.com
TTL: 3600
```

### Пример для разных платформ

#### Cloudflare
1. Зайдите в панель управления доменом
2. Перейдите в раздел DNS
3. Нажмите "Add record"
4. Выберите тип: CNAME
5. Name: roadmap
6. Target: ваш-основной-домен.com
7. Proxy status: Proxied (оранжевое облако)

#### Yandex Cloud DNS
1. Зайдите в консоль Yandex Cloud
2. Выберите ваш DNS-зону
3. Создайте новую запись
4. Тип: CNAME
5. Имя: roadmap
6. Значение: ваш-основной-домен.com

#### Другие DNS провайдеры
Принцип одинаковый для всех:
- Создайте CNAME запись
- Имя: roadmap
- Значение: ваш основной домен

### Проверка работы

После настройки DNS (может занять до 24 часов):

1. Откройте roadmap.ваш-домен.com
2. Должна загрузиться страница дорожной карты
3. Основной домен (ваш-домен.com) должен показывать страницу подбора команды

### Локальная разработка

Для тестирования поддомена локально:

1. Отредактируйте файл hosts:
   - Windows: `C:\Windows\System32\drivers\etc\hosts`
   - Mac/Linux: `/etc/hosts`

2. Добавьте строку:
   ```
   127.0.0.1 roadmap.localhost
   ```

3. Запустите проект: `bun run dev`

4. Откройте в браузере:
   - http://localhost:5173 → Страница команды
   - http://roadmap.localhost:5173 → Дорожная карта

### Поддержка

Если поддомен не работает:
1. Проверьте DNS записи (nslookup roadmap.ваш-домен.com)
2. Очистите кэш DNS (ipconfig /flushdns для Windows)
3. Подождите распространения DNS (до 24 часов)
4. Проверьте настройки хостинга/CDN
